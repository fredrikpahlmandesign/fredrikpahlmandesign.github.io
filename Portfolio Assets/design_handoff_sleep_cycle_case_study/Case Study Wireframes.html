Animation blocks

how to show motion without it feeling like homework
1Device-framed loop
loop
▣ autoplay · muted · loops
Use for micro-interactions — a toggle, a transition, a gesture. Short silent loop inside a phone; no controls, no sound. The cleanest way to prove "it moves".
2Full-bleed reel band
dark full-bleed reel — the showpiece motion moment
Use for the one hero animation per case. Dark band makes motion pop and gives the page a beat of drama. Pair with a one-line caption.
3Before → After
beforeafter
⇄
Use to show impact of a redesign — drag-to-reveal slider, or a simple tap-toggle. Instantly legible; great for "old vs new".
4Captioned sequence strip
frame 1

frame 4
done
Use when motion needs explaining — break an animation into key frames with captions. Works in print/PDF too, where video can't play.
5Live prototype embed
▶ interactive

embedded Figma / Framer / coded prototype
Use for the one thing worth touching. Powerful but heavy — gate behind a "Launch prototype" click so it never slows the page.
Rule of thumb across all of these: one motion showpiece per case, everything else static. Motion earns attention precisely because it's rare on the page.

Sleep Cycle — assembled

screen-heavy AND motion-heavy · paced so it breathes
The challenge you named: lots of screens and lots of motion. The fix isn't picking one — it's rhythm. Alternate a dense screen block with a single motion beat, so neither fatigues. Watch the left rail tags ↓ to see the pulse: screens → motion → screens → motion → showpiece.
